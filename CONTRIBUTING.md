# Contributing

Thanks for contributing. Please follow these guidelines to make reviews fast and predictable.

1. Fork and create a feature branch: `feature/<short-desc>`
2. Keep PRs small and focused.
3. Write a clear PR description and link issues.
4. Run linters and basic checks before opening PRs.

Coding standards
- Follow existing project style. Consider adding `eslint` and `prettier` for consistency.
- Use meaningful commit messages.

PR Review
- Add testing steps and a short summary of what you changed.
- Address reviewer comments promptly.
